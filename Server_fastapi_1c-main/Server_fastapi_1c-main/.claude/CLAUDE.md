# Workflow
1. Сначала анализируй задачу
2. Потом предложи план
3. Только потом пиши код

# Git Workflow

При изменении кода:

1. Проверь изменения:
   - git status
   - git diff

2. Создай коммит:
   - сообщение должно быть коротким и понятным
   - формат: <type>: <описание>
     примеры:
     - изменения: добавил эндпоинт регистрации
     - исправления: исправил ошибку подключения

3. Выполни команды:
   - git add .
   - git commit -m "<message>"

4. При необходимости:
   - git push origin main

# Rules

- Никогда не пушь без явного запроса пользователя
- Всегда показывай diff перед коммитом
- Не коммить временные файлы (.env, node_modules)
