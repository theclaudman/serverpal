# Service FastAPI 1С

Веб-сервис на FastAPI для визуализации данных из 1С:УНФ через OData API.  
Предоставляет три отчёта с авторизацией, тёмной темой и фильтрацией по периоду.

---

## Возможности

| Страница | Маршрут | Описание |
|---|---|---|
| Главная | `/` | Навигация по разделам |
| Прайс-лист | `/price-list` | Номенклатура с ценами, остатками и резервами |
| Дашборд менеджеров | `/dashboard/managers` | KPI по менеджерам за выбранный период |
| Отчёт по продажам | `/report/sales` | Детализация продаж по накладным |
| Авторизация | `/login` | Вход по логину и паролю |
| Регистрация | `/register` | Создание аккаунта (логин, пароль, IP и имя публикации 1С) |

---

## Стек

- **Python 3.12+**
- **FastAPI** — веб-фреймворк
- **python-multipart** — обработка форм
- **urllib** — HTTP-запросы к 1С OData (без сторонних зависимостей)
- **Chart.js** — графики в отчёте продаж
- **Bootstrap Icons** — иконки

## Установка и запуск

```bash
# Создать виртуальное окружение
python -m venv venv

# Активировать
venv\Scripts\activate          # Windows
source venv/bin/activate       # Linux / macOS

# Установить зависимости
pip install fastapi uvicorn python-multipart

# Запустить
python main.py
```

Сервис будет доступен по адресу: `http://localhost:8000`

## Конфигурация

Файл `config.py`:

```python
SECRET_KEY  = "change-me-in-production"   # ключ подписи сессионных cookie

# Ref_Key видов цен из Catalog_ВидыЦен
PRICE_COLUMNS = {
    "7481362d-...": "Розничная",
    "72e9e83e-...": "Оптовая",
}
```

> Имя публикации OData (`onec_publication`) больше не задаётся глобально в `config.py` — каждый пользователь указывает его при регистрации. URL к API строится динамически: `http://{server_ip}/{onec_publication}/odata/standard.odata`.

## Авторизация

### Регистрация (`/register`)

При первом использовании необходимо создать аккаунт. Потребуется ввести:

- **IP-адрес сервера 1С** — только хост, например `192.168.1.10` или `localhost`
- **Имя публикации 1С** — имя из настроек публикации веб-сервера, например `unf_dashboard`
- **Логин** — пользователь 1С
- **Пароль** — пароль пользователя 1С

Сервис проверяет подключение к 1С и сохраняет профиль в локальную SQLite-базу (`users.db`).  
После успешной регистрации выполняется автоматический вход.

### Вход (`/login`)

При повторных визитах достаточно ввести только **логин** и **пароль**.  
IP-адрес сервера 1С берётся из базы данных автоматически.

### Сессия

После входа создаётся подписанная cookie-сессия (HMAC-SHA256) сроком на **12 часов**.  
Пока сессия активна, повторный ввод данных не требуется.  
Учётные данные и имя публикации из cookie используются для каждого запроса к 1С.

## Интеграция с 1С

Сервис подключается к 1С через стандартный **OData REST API** (публикация базы на веб-сервере).

Используемые объекты:

| Объект 1С | Данные |
|---|---|
| `Catalog_Номенклатура` | Номенклатура и группы |
| `InformationRegister_ЦеныНоменклатуры` | Цены |
| `AccumulationRegister_ЗапасыНаСкладах` | Остатки |
| `AccumulationRegister_РезервыТоваровОрганизаций` | Резервы |
| `Document_ЗаказПокупателя` | Заказы покупателей |
| `Document_РасходнаяНакладная` | Расходные накладные |
| `Catalog_Сотрудники` | Сотрудники / менеджеры |
| `Catalog_Контрагенты` | Контрагенты |
| `AccumulationRegister_ДоходыИРасходы` | Выручка и себестоимость |
| `AccumulationRegister_РасчетыСПокупателями` | Оплаты и задолженности |
| `Document_Событие` | События CRM |
