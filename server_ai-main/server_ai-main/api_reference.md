# API Reference — 1C AI Bridge

## Общие правила

- Все запросы и ответы в формате **JSON**
- Каждый запрос содержит блок `credentials` для идентификации базы 1С
- `base_id` в ответах — автоматически вычисляемый хэш из `ip` + `login`

---

## Блок credentials (передаётся в каждом запросе)

```json
"credentials": {
    "login": "string",      // логин пользователя базы 1С
    "password": "string",   // пароль пользователя базы 1С
    "ip": "string"          // IP-адрес базы 1С (например: "192.168.1.10:8080")
}
```

---

## Эндпоинты сервиса

### POST /bases/register
Регистрирует базу 1С в реестре. При повторном вызове — обновляет данные.

**Запрос:**
```json
{
    "credentials": {
        "login": "admin",
        "password": "secret",
        "ip": "192.168.1.10:8080"
    },
    "display_name": "Моя база УНФ"   // опционально, человекочитаемое название
}
```

**Ответ:**
```json
{
    "base_id": "a1b2c3d4e5f6",
    "login": "admin",
    "ip": "192.168.1.10:8080",
    "display_name": "Моя база УНФ"
}
```

---

### GET /bases/
Возвращает список всех зарегистрированных баз (без паролей).

**Запрос:** без тела

**Ответ:**
```json
[
    {
        "base_id": "a1b2c3d4e5f6",
        "login": "admin",
        "ip": "192.168.1.10:8080",
        "display_name": "Моя база УНФ"
    }
]
```

---

### POST /report/daily
Принимает сырые данные из 1С за день, передаёт в GigaChat, возвращает и сохраняет ежедневный отчёт.

**Запрос:**
```json
{
    "credentials": {
        "login": "admin",
        "password": "secret",
        "ip": "192.168.1.10:8080"
    },
    "data": { }   // произвольная структура — сырые данные из 1С (объект или массив)
}
```

**Ответ:**
```json
{
    "base_id": "a1b2c3d4e5f6",
    "report_type": "daily",
    "report_date": "2026-04-12",
    "content": "## Ежедневный отчёт\n...",   // готовый отчёт в Markdown
    "saved_path": "data/connected_bases/a1b2c3d4e5f6/daily_reports/report_2026-04-12.md"
}
```

---

### POST /report/weekly
Принимает сырые данные из 1С за неделю, передаёт в GigaChat, возвращает и сохраняет еженедельный отчёт.

**Запрос:** — аналогично `/report/daily`

**Ответ:** — аналогично `/report/daily`, `report_type: "weekly"`

---

### POST /chat/
Принимает произвольный текстовый промпт, передаёт в GigaChat, возвращает ответ.

**Запрос:**
```json
{
    "credentials": {
        "login": "admin",
        "password": "secret",
        "ip": "192.168.1.10:8080"
    },
    "prompt": "Какова средняя выручка за последние 7 дней?"
}
```

**Ответ:**
```json
{
    "base_id": "a1b2c3d4e5f6",
    "answer": "На основе предоставленных данных средняя выручка составила..."
}
```

---

### POST /query/
Выполняет запрос на языке запросов 1С на целевой базе и возвращает результат.

**Запрос:**
```json
{
    "credentials": {
        "login": "admin",
        "password": "secret",
        "ip": "192.168.1.10:8080"
    },
    "query_text": "ВЫБРАТЬ Номенклатура.Наименование ИЗ Справочник.Номенклатура КАК Номенклатура"
}
```

**Ответ:**
```json
{
    "base_id": "a1b2c3d4e5f6",
    "result": { }   // данные, возвращённые базой 1С (структура зависит от запроса)
}
```

---

### GET /health
Проверка состояния сервиса.

**Ответ:**
```json
{
    "status": "ok"
}
```

## Запрос к базе 1С (исходящий)

Сервис отправляет запросы к базе 1С при вызове `/query/` и при формировании автоматических отчётов.

**Метод:** `POST`  
**URL:** `http://<ip>/hs/ai/query`  
**Авторизация:** `Basic Auth` (login + password из credentials)

**Заголовки:**
```
Content-Type: application/json
Authorization: Basic <base64(login:password)>
```

**Тело запроса:**
```json
{
    "query": "ВЫБРАТЬ ..."   // текст запроса на языке 1С
}
```

**Ожидаемый ответ от 1С:**
```json
{
    // произвольная JSON-структура с результатом выполнения запроса
}
```

> **Примечание для разработчика 1С:** на стороне базы должен быть реализован HTTP-сервис по пути `/hs/ai/query`, принимающий POST-запрос с полем `query`, выполняющий его и возвращающий результат в JSON.

## Автоматические отчёты (по таймеру)

Формируются без участия внешних запросов — планировщик запускает их самостоятельно.

| Тип | Расписание | Источник данных |
|---|---|---|
| Ежемесячный | 1-е число месяца, 06:00 | Все ежедневные отчёты прошлого месяца |
| Ежеквартальный | 1 янв / апр / июл / окт, 06:30 | Все ежемесячные отчёты прошлого квартала |

Результат сохраняется в:
```
data/connected_bases/<base_id>/monthly_reports/report_YYYY-MM-01.md
data/connected_bases/<base_id>/quarterly_reports/report_YYYY-MM-01.md
```
