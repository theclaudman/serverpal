# Описание сервиса 1C AI Bridge

## Назначение

Сервис является связующим звеном между базами данных 1С УНФ 3.0 и нейросетью GigaChat.
Принимает сырые данные из 1С, передаёт их в AI, возвращает готовые отчёты и аналитику.

## Архитектура

```
[1С УНФ 3.0] ──POST──► [FastAPI сервис] ──► [GigaChat API]
                              │
                              ▼
                     [Файловое хранилище]
```

## Компоненты

### API слой (`app/api/routes/`)
- `reports.py` — приём ежедневных и еженедельных отчётов от 1С
- `chat.py` — обработка произвольных текстовых запросов
- `bases.py` — регистрация и управление базами 1С

### Сервисный слой (`app/services/`)
- `ai_service.py` — интеграция с GigaChat через официальный SDK
- `storage_service.py` — сохранение и чтение отчётов из файловой системы
- `onec_service.py` — отправка запросов к базе 1С через urllib

### Ядро (`app/core/`)
- `config.py` — настройки через `.env` (pydantic-settings)
- `security.py` — идентификация баз по ip+login, реестр в JSON
- `scheduler.py` — APScheduler для ежемесячных и ежеквартальных отчётов

## Идентификация баз

Каждая база 1С идентифицируется по паре `ip + login`.
Из них генерируется `base_id` (MD5-хэш, первые 12 символов).
Под этим `base_id` создаётся папка в хранилище отчётов.

## Автоматические отчёты

| Тип | Расписание | Источник данных |
|---|---|---|
| Ежемесячный | 1-е число, 06:00 | Агрегация ежедневных отчётов прошлого месяца |
| Ежеквартальный | 1 янв/апр/июл/окт, 06:30 | Агрегация ежемесячных отчётов прошлого квартала |

Сырые данные из 1С для этих отчётов **не запрашиваются** — используются уже сохранённые отчёты нижнего уровня.

## Промпты

Системные промпты для AI хранятся в папке `prompts/` в виде `.txt` файлов.
Редактирование промптов не требует перезапуска сервиса.
